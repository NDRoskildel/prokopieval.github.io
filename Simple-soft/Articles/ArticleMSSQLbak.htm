В зависимости от используемой СУБД процесс резервирования базы данных несколько отличается.

СУБД Microsoft™ Access

Создание резервной копии базы данных производится непосредственно из программы через меню Файл - Создать резервную копию

В каталоге, заданном в настройках программы в качестве папки для резервных копий базы данных, будет создан файл с указанным именем.

СУБД Microsoft™ SQL SERVER

Создание резервной копии базы данных возможно несколькими способами:

  • В среде Microsoft™ SQL SERVER Managment Studio (предпочтительно)
  • Непосредственно в программе

Создание резервной копии базы данных в среде Microsoft™ SQL SERVER Managment Studio

Необходимо вызвать контекстное меню целевой базы данных и выбрать пункт Задачи - Создать резервную копию....

В открывшемся окне убеждаемся, что тип резервной копии - полная, а в нижней части окна будет автоматически указан полный путь и имя файла резервной копии. При необходимости путь и имя файла можно изменить, необходимо следить только, чтобы у учетной записи, под которой стартует служба SQL SERVER в системе, были достаточные полномочия для записи файлов в указанный каталог.

В каталоге, заданном в настройках программа в качестве папки для резервных копий базы данных или в каталоге резервных копий SQL SERVER, будет создан файл с указанным именем.

Создание резервной копии базы данных из программы

В данном случае создание резервной копии происходит точно также, как и в случае с СУБД Microsoft™ Access: через меню Файл - Создать резервную копию
